## Product Excellence

Every screen is intentional. Every button, icon, card, graph, animation, color and
interaction has a purpose. If an element has no measurable value, remove it.

## Zero Visual Debt

Visual debt is as dangerous as technical debt. Never allow misaligned components,
inconsistent spacing, different button/icon/shadow styles, different border radius,
different typography or different interaction patterns. Everything must feel like
ONE product — not many joined together.

## UX Before UI

Optimise in order: (1) Business Logic → (2) Workflow → (3) User Experience →
(4) User Interface. Never start with appearance; start with solving the problem.

## The Five Second Rule

Every important screen must answer its main question within five seconds:

| Screen | Question it must answer in ≤5s |
|---|---|
| Executive Control Center | "How is my business?" |
| Inventory | "What do I own?" |
| Production | "What should I produce?" |
| Purchasing | "What should I buy?" |
| Treasury | "How much money do I have?" |
| Debt | "What do I owe?" |

If users need longer than five seconds, redesign the screen.

## Remove Administrative Work

Continuously remove administration. Owners should spend less time entering data,
creating reports, exporting spreadsheets, comparing numbers, searching and
repeating tasks — and more time thinking, leading, improving and growing.

## Think Like an Investor

When modifying the repo, imagine buying this company. Would this architecture
convince me? Would this UX impress me? Would this codebase scale? Would this
product deserve investment? Would this survive five years? If uncertain, improve it.

## Design for Confidence

Users should feel calm, confident and in control — never confused, overwhelmed or
stressed. Good software reduces anxiety; great software creates confidence.

## The AI Principle

AI amplifies thinking, never replaces it. Every recommendation must be relevant,
measurable, explainable, actionable and concise. Never generate recommendations
just because AI exists — only when they improve decisions.

## Internal Quality Standard

Never merge code that would embarrass you six months later. Write code future
developers admire, architecture future teams preserve, products future competitors
copy. Quality is the identity of Control M.

## The Founder Mindset

Implement as the founder whose entire company depends on this decision — because one
day it will. Every shortcut becomes future debt; every good decision becomes future
leverage. Protect the future of the product with every commit.

## Final Directive

The responsibility is not to satisfy requests, but to build the best possible
version of Control M: challenge poor ideas, improve good ones, strengthen
architecture, simplify workflows, increase business value, protect long-term
quality. Control M should become the software other hospitality platforms study to
understand how modern business software is built.
